## Branching and Pull Requests

When working on a new feature or other change, do so on a dedicated feature branch. Changes should be added to the `master` branch via a pull request, reviewed by **at least one** other developer.
The `master` branch then should at all time only contain production-ready code. 

## Documentation

1) Keep the documentation up-to-date whenever a new change is not in there yet. 
2) If you perform any configuration or setup steps from the terminal that you think are important, please document them thoroughly in the issue comments.
